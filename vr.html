Virtual Reality Simulation of Magnetic Resonance Imaging

Duration: Jan 2020 - May 2020

Engagement Type: College Project

Team Size: 3

Checkout:

  1. Presentation Slides
  2. Code Repository

My role: I was responsible for creating mockups, storyboarding, designing and animating 3D models. The programming work was majorly done by my teammates.

Objective

E-learning or technology-based learning is becoming more and more common these days. Video-based learning solutions provide students with a visual approach to the concepts, increasing their retention power while simultaneously making learning a more fun experience. Two-dimensional videos can give students a fair bit of idea of how something looks, but they lack immersion. This is where Virtual Reality(VR) based applications can become very useful. VR apps can help to showcase objects and phenomenon which are too large or too small for the naked eyes. Most importantly, there's the factor that VR apps can be interacted with, making the learning process more engaging. Imagine students are being taught about volcanic eruptions in a virtual classroom. They can use the immersive module to go below the earth's surface, get a sense of what it is to be magma and experience different factors that lead to the eruptions. This way they can explore, identify, and at-times experiment with the factors that lead to eruptions. For this project, we set out to build a VR app which illustrates the process of Magnetic Resonance Imaging(MRI) scanning in a simulated classroom environment. MRI scanning being a nuclear level phenomenon is difficult to understand through text or video alone. Students don't usually have access to MRI machines either to see how they operate. But these problems don't exist in the virtual world.

Deliverables

Developing a VR application to learn about MRI, which contains the following sections:

  1. Learning about the theory of MRI.
  2. Experiencing MRI machine, its different components and real world usage.
  3. Showcasing how the phenomenon unfolds at the atomic level.

Process

The project had two phases - designing and development.

Designing

The team started by exploring different VR alternatives. Eventually, we decided to use Google Cardboard as it was the most affordable solution (approx. $4) in the market and thus ideal for students. Also, the only needed hardware to get it working was an Android Phone, which most of the students we interviewed used.

The workflow we followed had three steps. The first step was to storyboard the whole project. This helped us to communicate ideas and expectations within the team. The second step was brainstorming, creating mockup (sketches on paper), getting other students' feedback and iterating until we got satisfied with the designs. The third step was developing the application.

Models imported to create the classroom scene

Google provides Unity Plugin that helps us to easily and efficiently set up everything required for creating VR experiences in Unity. Some useful components from the plugin include GvrAudio, GvrViewer, GvrRecticlePointer, GvrViewer, GvrAudioRoom.

Secondly, we added behaviour to all the different models and made them interactive.

Adding motion to classroom fans.

Adding event trigger to the projector button.

Script flow or the program flow goes as - the first user uses the PLAY button (ProjectorSwitch.cs), video runs and gets over, and all the chairs and tables disappear from screen (afterProjection.cs), then chair goes back (chairMove.cs) and then all possible camera views (CameraView.cs) are shown from which user can see the models. These are the main scripts that run the program, apart from these other small scripts are used to add behaviour to elements like rotating the fan etc.

Conclusion

Sequence 1 : Once you wear the VR headset, you find yourself in a classroom. You can see a projector that is interactable and has a play button.

Sequence 2 : You use the Play button to start the projector. A short video lecture on MRI starts with 3D audio.

Sequence 3 : Once the lecture is over, all furniture disappears to make space in the classroom, except your chair which move to the back of classroom.

Sequence 4 : Camera View button appears above the chair prompting user to go to it.

Sequence 5 : Going into the camera view, the user can view the MRI machine, the coil inside it and other internals.

Sequence 6 : In a similar way, the user goes through different sequences, where he can interact and experience MRI at different levels.

After finishing the project, we shared it with some of our friends and informally asked about their experiences. Most of them loved the application. One of them recounted, "to see the machine go through the process helped me understand the topic clearly". However, another friend expressed disapproval by saying, "I felt claustrophobic while using the application".

Reflections

  1. We should have performed evaluations after completing the project. We could have used two groups of students, the first group would learn the topic traditionally and the second group would learn using the VR application. Then both the groups would give a test. The difference in performance on test would have indicated if the project was actually useful or not.
  2. Having multiple students attend the lecture at the same time, making systems more interactive so that students can control more variables in the system.
