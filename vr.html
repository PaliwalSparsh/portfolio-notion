Exploration of Virtual Reality in Education

Duration: Jan 2017 - May 2017

Engagement Type: College Project

Team Size: 3 (Sandeep Pandey, Sparsh Paliwal and Varun Rao)

Checkout:

  1. Code Repository
  2. Presentation Slides

My role: I was responsible for creating mockups, storyboarding, designing and animating 3D models. The programming work was majorly done by my teammates, I was responsible for code reviews.

Objective

E-learning or technology-based learning is becoming more and more common these days. Video-based learning solutions provide students with a visual approach to the concepts but they lack immersion. This is where Virtual Reality(VR) based applications can become very useful. VR apps can help to showcase objects and phenomenon which are too large or too small for the naked eyes. Most importantly, VR apps can be interacted with, making the learning process more engaging. Imagine students are being taught about volcanic eruptions in a virtual classroom. They can use the immersive module to go below the earth's surface, get a sense of what it is to be magma and experience different factors that lead to eruptions. This way they can explore, identify, and at-times experiment with the factors that lead to eruptions.

For this project, we set out to build a VR app that illustrates the process of Magnetic Resonance Imaging(MRI) scanning in a simulated classroom environment. MRI scanning being a nuclear level phenomenon is difficult to understand through text or video alone. Students don't usually have access to MRI machines either to see how they operate. But these problems don't exist in the virtual world.

Deliverables

Developing a VR application to learn about MRI, which contains the following sections:

  1. Learning about the theory of MRI.
  2. Experiencing the MRI machine, its different components and real-world usage.
  3. Showcasing how the phenomenon unfolds at the atomic level.

Process

The project had two phases - designing and development.

Designing

The team started by exploring different VR alternatives. Eventually, we decided to use Google Cardboard as it was the most affordable solution (approx. $4) in the market and thus ideal for students. Also, the only needed hardware to get it working was an Android Phone, which most of the students we interviewed used.

The workflow we followed had three steps. The first step was to storyboard the whole project. This helped us to communicate ideas and expectations within the team. The second step was brainstorming, creating a mockup (sketches on paper), getting other students' feedback and iterating until we got satisfied with the designs. The third step was developing the application.

Storyboard for the project

Classroom design - learning MRI theory

Here are few things that we considered to design good user experience for users inside VR: (1) using motion of objects in environment to direct user's attention to the next possible action, (2) adding spacial audio to improve immersion, (3) using a reticle to simplify the task of interacting with elements, (3) addressing the text readability issues (due to low resolution and aliasing in VR using mobile phones) with proper text contrast and size properties and (4) using traditional UX principles like Fitt's law for target size.

Development

The tools we used to develop the project were -

  1. Cinema 4D to create 3D models and animations.
  2. Unity 3D to program the models and add behavior.

Work in Unity 3D

Work done in Unity is distributed over GUI and code. We’ll go over different sections we worked on in the project to reach the final goal.

Firstly, we imported all the models in Unity and then arranged them to make the classroom.

Models imported to create the classroom scene

Google provides Unity Plugin that helps us to easily and efficiently set up everything required for creating VR experiences in Unity. Some useful components from the plugin include GvrAudio, GvrViewer, GvrRecticlePointer, GvrViewer, GvrAudioRoom.

Secondly, we added behavior to all the different models and made them interactive.

Adding event trigger to the projector button.

Adding motion to classroom fans.

Script flow - user uses the PLAY button present on the projector (projectorSwitch.cs) and the video explaining MRI theory gets played. Once the video finishes, chairs and tables present in the room disappear (afterProjection.cs), except for one chair that goes to the backside of the classroom (chairMove.cs). After the user reaches the chair, a model of MRI machine appears in front of him which he can observe from different camera angles (cameraView.cs). Then the user is presented with a series of models(MRI machine internals, MRI at the atomic scale). Other scripts are used to add behavior to elements like rotating the fan etc.

You can check out the code repository here.

Final Application

Sequence 1: Once you wear the VR headset, you find yourself in a classroom. You can see a projector that is interactable and has a play button.

Sequence 2: You use the Play button to start the projector. A short video lecture on MRI starts with 3D audio.

Sequence 3: Once the lecture is over, all furniture disappears to make space in the classroom, except your chair which moves to the back of classroom.

Sequence 4: Camera View button appears above the chair prompting user to go to it.

Sequence 5: The user is presented with a series of models. The user can view the MRI machine, its coil and other internals.

Sequence 6: In a similar way, the user goes through different sequences, where he can interact and experience MRI at different levels.

Conclusions

We were able to meet the deliverables we set out to achieve. It was satisfying to see our imagination come to fruition. After finishing the project, we shared it with some of our friends and informally asked about their experiences. Most of them loved the application. One of them recounted, "to see the machine go through the process helped me understand the topic clearly". However, another friend expressed disapproval by saying, "I felt claustrophobic while using the application".

The project helped us realise that VR is a useful tool for education. The advancements taking place in this field are making such devices more powerful and cheaper at the same time.

Reflections

  1. We should have performed evaluations after completing the project. We could have used two groups of students, the first group would learn the topic traditionally and the second group would learn using the VR application. Then both the groups would give a test. The difference in performance on test would have indicated if the project was successful or not.
  2. Having multiple students attend the lecture simultaneously to bring the environment closer to reality.
  3. Attaching more variables to models so that students have more room for experimentation.
