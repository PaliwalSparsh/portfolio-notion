Component Library

Duration: September 2021

Engagement Type: College Project

Team Size: 1

Checkout:

  1. Figma File

My role: ..............

Objective

As a part of the Interaction Design Course(711) at University of Maryland. We were required select an imaginary application which would be useful for 10-12 of my close friends. The idea which I selected was –

An app that helps student find places to study(indoor and outdoor) throughout the campus.

Deliverables

  1. Component Library
  2. Screens for the app made using the components.
  3. Use this application along with your friends to create a video.

Process

The idea was to create components based on the requirements of the project. This meant, I had to first layout the project requirements. For this –

  1. Performed a user interview to learn about user problems and understand even if this was a problem.
  2. Laying out the information architecture.
  3. Creating desktop and mobile screen sketches.
  4. Designing the component library based on the requirement.
  5. Assembling the screens.

User Interview

Based on the interviews I conducted I was able to confirm that finding places to study was indeed a problem. With my learnings from the interview, I laid out the requirements for the product –

Further, to make the product feasible for business. I introduced paid advertisements for study spaces such as cafes.
